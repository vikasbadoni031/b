Golang CLI plugin to extend kubectl functionality

2024

Extending the functionality of kubectl by creating a custom plugin using the popular Go package, Cobra. Kubernetes offers extensive command-line tools, but sometimes you need additional commands tailored to your specific workflow. By using Go and Cobra, you can easily add your own commands to kubectl, helping with automating repetitive tasks.

This app is more foucsed on how to use cobra package and it focuses on implemnting a simple usecase of getting the deploymnets based on users input parameter. Based on input this pulls up matching depoloymnet from the cluster and displays requested memory and cpu, “None” in case no requests are set. This will help to easily identify problmetic deployments in the system. The funtionality can be easily extended to any other types, objects and for custom requiremtes.

// packages used
"k8s.io/client-go/kubernetes"
"k8s.io/client-go/tools/clientcmd"
"github.com/spf13/cobra"

With in app we are using cleintcmd to get the requiored clientset to provide cluster authentication.

clientset, err := kubernetes.NewForConfig(config)

Then use this cleintset to use the List method from DeploymentInterface using AppsV1Interface from the kubernetes client go package.

https://pkg.go.dev/k8s.io/client-go/kubernetes/typed/apps/v1#AppsV1Interface

https://pkg.go.dev/k8s.io/client-go/kubernetes/typed/apps/v1#DeploymentInterface

clientset := client.ClientSet()
deployments, err := clientset.AppsV1().Deployments(opt.namespace).List(context.TODO(), metav1.ListOptions{})

once we have the deployment details we can loop through them and provide our custom logic to get, modify them. Here we loop through all the containers and get their requests for cpu and memory which is then displayed by the client.

This app currently takes deploymnet name as a input to macth and -n as a flag for namespace input which defaults to “default.
